# 📘 3.1 Multi-Domain Trust Analyzer Script

## 📋 Overview

**Purpose:** Analyze and visualize Active Directory forest and domain trust relationships, verify cross-domain authentication, and assist with trust management.

| Property      | Value                                                                |
| ------------- | -------------------------------------------------------------------- |
| 🎯 Purpose    | Analyze and visualize trust relationships across forests and domains |
| 📈 Difficulty | Tough                                                                |

# 📄 Script Code

```powershell
# L3.1: Multi-Domain Trust Analyzer Script
# ============================================

function Get-ForestTrustAnalysis {
    param([string]$RootDomain)

    Write-Host "=== Forest Trust Analysis ===" -ForegroundColor Cyan

    try {
        $Forest = Get-ADForest -Identity $RootDomain

        Write-Host "Forest Root: $($Forest.RootDomain)"
        Write-Host "Domains: $($Forest.Domains -join ', ')`n"
    }
    catch {
        Write-Host "7 Cannot access forest: $_" -ForegroundColor Red
        return
    }

    # Get all domains and their trusts
    $TrustMap = @{}

    foreach ($Domain in $Forest.Domains) {

        Write-Host "Domain: $Domain" -ForegroundColor Yellow

        try {

            $DomainObj = Get-ADDomain -Identity $Domain

            $Trusts = Get-ADTrust -Filter * -Server $Domain -ErrorAction SilentlyContinue

            if ($Trusts) {

                Write-Host " Trusts:" -ForegroundColor Cyan

                foreach ($Trust in $Trusts) {

                    $Type = switch ($Trust.TrustType) {
                        "External" { "External Trust" }
                        "Forest" { "Forest Trust" }
                        "Downlevel" { "NT Domain Trust" }
                        "DOWNLEVEL" { "NT Domain Trust" }
                        default { $Trust.TrustType }
                    }

                    $Direction = switch ($Trust.TrustDirection) {
                        "Bidirectional" { "« Bidirectional" }
                        "Inbound" { "¬ Inbound" }
                        "Outbound" { "® Outbound" }
                        default { $Trust.TrustDirection }
                    }

                    Write-Host " $Direction : $($Trust.Name) ($Type)"
                }
            }
            else {

                Write-Host " No trusts configured" -ForegroundColor Gray
            }
        }
        catch {

            Write-Host " 7 Error: $_" -ForegroundColor Red
        }

        Write-Host ""
    }
}

function Test-CrossDomainAuth {
    param(
        [string]$SourceDomain,
        [string]$TargetDomain,
        [string]$TestUser
    )

    Write-Host "`n=== Cross-Domain Authentication Test ===" -ForegroundColor Cyan

    try {

        # Check if user from source domain can auth to target
        $SourceUser = Get-ADUser -Identity $TestUser -Server $SourceDomain
        $TargetForest = Get-ADForest -Identity $TargetDomain

        Write-Host "Source User: $($SourceUser.UserPrincipalName)"
        Write-Host "Target Domain: $TargetDomain"

        # Verify trust relationship
        $DomainTrust = Get-ADTrust -Filter { Target -eq $TargetDomain } `
            -Server $SourceDomain -ErrorAction SilentlyContinue

        if ($DomainTrust) {

            Write-Host "3 Trust relationship exists" -ForegroundColor Green
            Write-Host " Direction: $($DomainTrust.TrustDirection)"
            Write-Host " Type: $($DomainTrust.TrustType)"
        }
        else {

            Write-Host "7 No trust relationship found" -ForegroundColor Red
        }
    }
    catch {

        Write-Host "7 Error: $_" -ForegroundColor Red
    }
}

function New-ADForestTrust {
    param(
        [string]$LocalDomain,
        [string]$RemoteDomain,
        [string]$RemoteAdminCred,
        [string]$TrustType = "Forest"
    )

    Write-Host "Creating $TrustType trust: $LocalDomain <-> $RemoteDomain" `
        -ForegroundColor Yellow

    try {

        # This requires proper credentials and coordination
        New-ADTrust -Name $RemoteDomain -TrustType $TrustType `
            -Direction Bidirectional -Server $LocalDomain

        Write-Host "3 Trust created successfully" -ForegroundColor Green
    }
    catch {

        Write-Host "7 Error creating trust: $_" -ForegroundColor Red
    }
}
```

# ⚙️ Functions Included

## 🌲 1. Forest Trust Analysis

**Function:** `Get-ForestTrustAnalysis`

Retrieves all domains within an Active Directory forest and enumerates trust relationships for each domain.

### Parameter

| Parameter    | Description        |
| ------------ | ------------------ |
| `RootDomain` | Forest root domain |

### Analysis Includes

* 🌲 Forest root information
* 🌐 Domain inventory
* 🤝 Trust relationships
* 🔄 Trust direction
* 🏷️ Trust type

---

## 🔑 2. Cross-Domain Authentication Test

**Function:** `Test-CrossDomainAuth`

Verifies whether a trust relationship exists between two domains and retrieves trust information for a specified user.

### Parameters

| Parameter      | Description                      |
| -------------- | -------------------------------- |
| `SourceDomain` | Source domain                    |
| `TargetDomain` | Target domain                    |
| `TestUser`     | User account used for validation |

### Verification Includes

* 👤 Source user lookup
* 🌐 Target domain validation
* 🤝 Trust existence
* 🔄 Trust direction
* 🏷️ Trust type

---

## 🤝 3. Create Forest Trust

**Function:** `New-ADForestTrust`

Creates a bidirectional trust relationship between two Active Directory domains.

### Parameters

| Parameter         | Description                      |
| ----------------- | -------------------------------- |
| `LocalDomain`     | Local Active Directory domain    |
| `RemoteDomain`    | Remote Active Directory domain   |
| `RemoteAdminCred` | Remote administrator credentials |
| `TrustType`       | Trust type (default: Forest)     |

# 🚀 Usage Examples

### Analyze Forest Trust Relationships

```powershell
Get-ForestTrustAnalysis -RootDomain "yourdomain.com"
```

### Verify Cross-Domain Authentication

```powershell
Test-CrossDomainAuth `
    -SourceDomain "domain1.com" `
    -TargetDomain "domain2.com" `
    -TestUser "jdoe"
```

### Create a Forest Trust

```powershell
New-ADForestTrust `
    -LocalDomain "yourdomain.com" `
    -RemoteDomain "partner.com"
```
